В этом проекте ML находится в `backend/analytics-service/main.py` и состоит из двух частей:

- **Модель прогноза урожайности** (реально ML)
- **Рекомендации по поливу** (в основном rule-based логика)

### 1 Основная ML-модель (урожайность)

Это `YieldPredictionModel`:

- алгоритм: `RandomForestRegressor` в `Pipeline` со `StandardScaler`;
- параметры леса:
  - `n_estimators=100`
  - `max_depth=8`
  - `min_samples_split=5`
  - `random_state=42`
- модель обучается **при старте сервиса** на синтетических данных (`_train_with_synthetic_data()`).

Фичи для прогноза:

- температура
- влажность воздуха
- осадки за 7 дней
- средняя температура за 7 дней
- влажность почвы
- солнечная радиация
- скорость ветра
- код культуры
- код типа почвы

Целевая переменная:
- абсолютная урожайность `yield` (т/га), которая в синтетике генерируется из агрономических эвристик.

---

### 2 Как считается confidence и диапазон прогноза

После предсказания:

- берутся предсказания отдельных деревьев (первые ~20),
- считается стандартное отклонение,
- строится интервал `yieldMin/yieldMax`,
- confidence определяется по коэффициенту вариации:
  - LOW / MEDIUM / HIGH.

То есть confidence тут не вероятностная калибровка, а инженерная оценка “стабильности” предсказаний леса.

---

### 3 Источник данных для инференса

Для реального запроса прогноза:

- сервис пытается подтянуть контекст поля из `field-service`,
- погоду — из Open-Meteo,
- если не удалось — использует fallback значения.

Эндпоинт прогноза:
- `POST /forecast/yield`

Также есть Kafka-flow:
- слушает `weather-data`,
- строит прогноз,
- публикует в `forecast-results`.

---

### 4 Метрики модели

Есть эндпоинт:
- `GET /model/metrics`

Он считает MAE/RMSE/R² и accuracy@15% на тест-сплите, но важно:
- тест и train тоже из **синтетики**, не из production-истории.
- То есть метрики полезны для демо/валидации пайплайна, но не как “боевой KPI качества на реальных данных”.

---

### 5 Важный вывод по зрелости

Текущая модель — хорошая **демо/PoC реализация**:
- понятные фичи,
- прогноз + факторы влияния,
- confidence,
- API + Kafka интеграция.

Но это еще не “промышленная” модель в строгом смысле, потому что:
- обучение синтетическое,
- нет явного offline/online retraining на реальных исторических данных,
- нет версионирования датасета/модели и MLOps-контура.

**Тестирование и метрики:** см. `techno-demo/docs/ML_TESTING.md` и скрипт `techno-demo/backend/analytics-service/scripts/ml_evaluation_suite.py`.
